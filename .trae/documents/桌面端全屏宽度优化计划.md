# 桌面端全屏宽度优化计划

## 问题描述

用户反馈：电脑端页面宽度没有和电脑屏幕一致，内容被限制在较窄的范围内，没有充分利用大屏空间。

## 根本原因分析

### 1. `src/styles/base.css` - 全局容器宽度限制
- **行226-245**：`#app` 在 `@media (min-width: 1024px)` 下虽然设置 `max-width: 100%`，但在 `@media (min-width: 1440px)` 被覆盖为 `max-width: 1400px` + `margin: 0 auto`，导致大屏下应用内容不占满屏幕

### 2. `src/pages/index/index.vue` - 首页内容宽度限制
- **行1989-2205**：`.content`、`.page-wrapper`、`.banner-section`、`.quick-actions` 等元素在不同断点被限制为 `max-width: 500px` → `600px` → `900px` → `1000px`
- 即使屏幕很大，内容也被限制在中间区域

### 3. `src/pages/activities/detail.vue` - 活动详情页
- **行992-1003**：内容区域被限制为 `max-width: 768px` → `800px`

### 4. `src/pages/ai-helper/detail.vue` - 互助详情页
- **行634-651**：内容区域被限制为 `max-width: 640px`

## 修复方案

### 核心思路

**桌面端（≥1024px）：内容占满可用宽度**
- 移除 `#app` 的 `max-width: 1400px` 限制，改为 `max-width: 100%`
- 移除各页面内部容器过窄的 `max-width` 限制（在桌面断点下改为更大的值）
- 保持合理的阅读宽度（不超过 `1200px - 1400px`），但显著放宽现有限制

### 具体修改

#### 1. `src/styles/base.css`
- **修改 `@media (min-width: 1440px)` 区块（行237-245）**：
  - `#app`: `max-width: 100%`（移除 `1400px` 限制）
  - 移除 `margin: 0 auto`

#### 2. `src/pages/index/index.vue`
- **修改桌面断点响应式样式（行1989-2205）**：
  - `.content`: `max-width: 1200px`（从 900px → 1200px）
  - `.page-wrapper`: `max-width: 1200px`（从 900px → 1200px）
  - `.banner-section`: `max-width: 1200px`（从 900px → 1200px）
  - `.quick-actions`: `max-width: 1200px`
  - `.feed-list`: `max-width: 1200px`
  - `.activity-scroll`: `max-width: 1200px`

#### 3. `src/pages/activities/detail.vue`
- **修改桌面断点响应式样式（行992-1003）**：
  - 内容容器 `max-width: 1000px`（从 768/800px → 1000px）

#### 4. `src/pages/ai-helper/detail.vue`
- **修改桌面断点响应式样式（行634-651）**：
  - 内容容器 `max-width: 1000px`（从 640px → 1000px）

## 预期效果

- 桌面端（≥1024px）：左侧导航栏 + 右侧内容区，内容占满屏幕可用空间
- 大屏（≥1440px）：应用整体宽度不再被 1400px 限制，而是真正充满整个视口
- 内容保持合理的阅读宽度（最大 1200px），避免文本过长难以阅读
- 整体视觉效果更加"桌面友好"，充分利用大屏幕空间

## 修改文件清单

| 文件 | 修改内容 | 优先级 |
|------|---------|--------|
| `src/styles/base.css` | 移除 #app 的 1400px 限制 | 高 |
| `src/pages/index/index.vue` | 放宽各内容容器的 max-width | 高 |
| `src/pages/activities/detail.vue` | 放宽内容容器的 max-width | 中 |
| `src/pages/ai-helper/detail.vue` | 放宽内容容器的 max-width | 中 |

## 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 文本过长影响阅读 | 低 | 保留合理的最大宽度（1000-1200px）|
| 移动端显示受影响 | 无 | 仅修改桌面端断点样式 |
| 元素对齐问题 | 低 | 统一调整 padding/margin |
