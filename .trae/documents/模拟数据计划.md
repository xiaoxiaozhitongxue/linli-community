# 模拟数据计划

## 问题概述

用户希望恢复模拟数据（帖子、活动、互助、消息等），并将这些模拟数据关联到测试账号，方便测试。

## 当前状态分析

1. **api.ts**: `activitiesApi.getActivities()` 返回空数组，`createActivity()` 保存到 localStorage
2. **ai-helper/index.vue**: 移除了默认模拟数据，新用户显示空状态
3. **profile/index.vue**: 从 localStorage 读取任务统计
4. **messages/index.vue**: 有模拟消息数据

## 修复方案

### 核心策略

采用混合策略：
- **默认显示模拟数据**：当 localStorage 无数据时
- **用户数据优先**：当 localStorage 有数据时，优先显示用户数据
- **支持用户追加**：用户可以在模拟数据基础上添加新内容

### 1. 修改 api.ts - 活动和帖子

**文件**: `src/utils/api.ts`

**修改内容**:
- `activitiesApi.getActivities()`: 优先从 localStorage 读取，无数据返回 `mockActivities`
- `activitiesApi.createActivity()`: 保存到 localStorage 时与模拟数据合并
- `postsApi.getPosts()`: 优先从 localStorage 读取，无数据返回 `mockPosts`
- 添加 `postsApi.createPost()` 支持保存用户帖子

### 2. 修改 ai-helper/index.vue - 互助任务

**文件**: `src/pages/ai-helper/index.vue`

**修改内容**:
- 恢复 `defaultTasks` 作为默认数据
- `loadFromStorage()`: localStorage 无数据时返回 `defaultTasks`
- 保存时：用户新任务添加到模拟数据前面

### 3. 修改 profile/index.vue - 任务统计

**文件**: `src/pages/profile/index.vue`

**修改内容**:
- 统计逻辑保持不变（从 localStorage 读取）
- 确保与 `ai-helper/index.vue` 的数据保持一致

### 4. messages/index.vue - 消息

**文件**: `src/pages/messages/index.vue`

**修改内容**:
- 保留现有模拟消息数据
- 无需修改

## 修复文件清单

| 序号 | 文件 | 修改内容 | 优先级 |
|------|------|----------|--------|
| 1 | src/utils/api.ts | 活动和帖子 API 恢复模拟数据支持 | P0 |
| 2 | src/pages/ai-helper/index.vue | 恢复互助任务默认模拟数据 | P0 |

## 验证步骤

1. 清除浏览器 localStorage
2. 登录测试账号
3. 验证：
   - 首页显示模拟帖子
   - 互助广场显示模拟任务
   - 活动中心显示模拟活动
   - 消息中心显示模拟消息
4. 发布一个新互助任务
5. 验证：新任务显示在列表顶部，模拟数据仍然保留

## 假设和决策

- **测试账号**：使用系统默认登录的用户
- **数据混合策略**：模拟数据作为基础，用户数据追加在前面
- **持久化**：用户新增数据保存到 localStorage
