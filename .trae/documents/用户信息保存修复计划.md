# 邻里社区APP - 用户信息保存和登录问题修复计划

## 📋 问题总结

### 问题1：修改个人信息后无法保存
- **表现**：编辑资料页面点击保存后，修改的信息没有保存成功
- **根本原因**：
  - API调用在MOCK模式下，只是更新了内存中的mockUsers数组
  - 没有同步到localStorage中保存的用户信息
  - edit.vue中调用updateProfile后，返回的数据与store中的数据不同步

### 问题2：登录后信息没变化
- **表现**：修改了个人信息后重新登录，显示的还是旧信息
- **根本原因**：
  - 登录时使用的是MOCK_MODE，authApi.login每次都创建一个新的模拟用户
  - 登录时没有从localStorage读取之前保存的用户信息
  - store中的setUser函数会将新的模拟用户覆盖localStorage中的旧用户

login: (data) => {

```typescript

updateProfile: (data) => {
  if (MOCK_MODE) {
    // 只更新内存中的mockUsers[0]

    resolve(mockUsers[0])  // 问题：没有同步到localStorage
  }
}
```

## 🛠️ 修复方案

### 方案：改进MOCK_MODE的数据持久化

#### 修改1：改进authApi.login
**文件**: `src/utils/api.ts`
**修改内容**:
- 在MOCK模式下，检查localStorage中是否有已保存的用户信息
- 如果有，使用已保存的用户信息（更新nickname等字段）
- 如果没有，创建新用户并保存到localStorage

#### 修改2：改进userApi.updateProfile
**文件**: `src/utils/api.ts`
**修改内容**:
- 更新mockUsers后，同步更新localStorage中的用户信息
- 返回更新后的用户信息

#### 修改3：确保store同步
**文件**: `src/store/index.ts`
**修改内容**:
- updateUser函数已经正确地将更新保存到localStorage
- 确保login后setUser正确保存用户信息

## 📝 详细实现步骤

### 步骤1：修改authApi.login
在MOCK_MODE下，改进登录逻辑：
1. 从localStorage读取已保存的用户信息
2. 如果存在，更新其nickname和community字段
3. 如果不存在，创建新用户并保存到localStorage

在MOCK_MODE下，改进更新逻辑：
1. 从localStorage读取当前用户信息
2. 合并新数据到用户对象
3. 保存更新后的用户信息到localStorage
4. 返回更新后的用户信息
