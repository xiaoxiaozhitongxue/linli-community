# 橙色区域闪烁 & 互助任务详情页加载问题 - 修复规格

## 问题概述

### BUG 1: 首页和互助页面橙色区域仍然闪烁错位
- **现象**: 用户反馈橙色区域（顶部状态栏/header）还是会错位闪烁
- **影响范围**: `src/pages/index/index.vue`, `src/pages/ai-helper/index.vue`
- **根本原因**: CSS变量 `--status-bar-height` 在 `:root` 中未定义，导致使用回退值时可能产生布局抖动

### BUG 2: 互助任务详情页一直显示加载中
- **现象**: 从互助任务列表页点击任务后进入详情页，页面一直显示加载中，无法显示任务内容
- **影响范围**: `src/pages/ai-helper/detail.vue`
- **根本原因**: 需要排查 `fetchTask` 函数执行流程

## 修复方案

### 修复 1: 橙色区域闪烁问题
1. 在 `src/styles/base.css` 的 `:root` 中添加 `--status-bar-height: 20px` 定义
2. 确保首页和互助页面使用固定的padding值而不是依赖可能未加载的CSS变量
3. 检查是否有其他动态计算的样式导致重排

### 修复 2: 互助任务详情页加载问题
1. 在 `detail.vue` 的 `fetchTask` 函数中添加详细的日志输出
2. 确保API调用失败时能正确回退到localStorage
3. 确保 `loading.value = false` 无论成功或失败都会执行

## 验证标准
| 验证点 | 预期结果 |
|---|---|
| 首页加载时橙色状态栏 | 无错位、无闪烁 |
| 互助页面加载时header | 无错位、无闪烁 |
| 点击互助任务进入详情页 | 3秒内显示任务详情或错误提示 |
| npm run build | 编译通过 |

## 测试步骤
1. 打开首页 → 检查顶部橙色区域加载是否正常
2. 进入互助页面 → 检查顶部橙色header加载是否正常
3. 发布一个新任务 → 点击进入详情页 → 检查是否正常显示
4. 从任务列表页点击任意任务 → 检查详情页是否正常加载
