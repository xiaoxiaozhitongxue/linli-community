# 邻里社区APP - 项目修复与重构

## 概述

**问题**: 当前项目存在严重的架构问题，页面功能无法正常工作，点击无响应，依赖缺失，且不是真正的跨平台兼容项目。

**目标**: 
1. 将项目重构为真正的跨平台 Vue 3 + Vite 项目（兼容 Web 端和移动端）
2. 修复所有功能模块，确保交互正常
3. 完善前后端对接
4. 部署到 Cloudflare Pages

**目标用户**: 
- 电脑 Web 端用户
- 手机 Web 端用户
- 社区居民

---

## 目标

1. **核心修复**: 使所有页面和功能可以正常交互
2. **跨平台兼容**: 一套代码，同时适配 Web 端（电脑+手机）
3. **功能完善**: 实现 PRD 中的核心功能模块
4. **部署就绪**: 可以正常部署到 Cloudflare Pages

---

## 非目标（暂不包含）

- 原生 App 打包（只做 Web 端）
- 复杂的数据库集成（先用 mock 数据演示）
- 支付功能

---

## 背景与上下文

当前问题:
1. 使用了 `uni.` 系列 API，但项目没有 UniApp 框架支持
2. 使用了 `<view>`、`<text>` 等自定义元素，但没有相应的组件库支持
3. API 请求和状态管理有问题
4. 路由和页面跳转不工作
5. 项目结构混乱（有两个 pages.json、两个 pages 文件夹）

---

## 功能需求

### FR-1: 项目架构重构
- 移除 UniApp 依赖，转为标准 Vue 3 + Vite 项目
- 使用标准 HTML 元素代替 UniApp 组件
- 使用 Vue Router 进行路由管理
- 使用 Pinia 或简单响应式状态管理
- 添加 Tailwind CSS 用于跨平台样式适配

### FR-2: 首页功能修复
- 轮播图正常显示和切换
- 快捷入口点击可以跳转
- 动态 feed 可以正常展示
- 点赞、评论、分享功能正常
- 下拉刷新、上拉加载更多

### FR-3: 登录/认证功能
- 手机号+验证码登录（演示用）
- 用户信息存储和获取
- 登录状态持久化

### FR-4: 邻里空间模块
- 社区客厅页面
- 活动中心页面
- 兴趣小组页面

### FR-5: AI互助模块
- 任务广场
- 任务详情
- 发布任务

### FR-6: 个人中心
- 个人信息展示
- 我的发布
- 设置页面

## 验收标准

### AC-1: 项目可以正常构建和运行
- **给定**: 完成项目重构
- **当**: 运行 `npm run dev` 和 `npm run build`
- **则**: 项目可以正常启动和构建，没有错误
- **验证类型**: 程序验证

### AC-2: 首页功能正常
- **给定**: 用户访问首页
- **当**: 点击轮播图、快捷入口、feed 卡片、点赞、评论等
- **则**: 相应功能正常响应，有视觉反馈
- **验证类型**: 人工判断

### AC-3: 页面路由正常
- **给定**: 用户在任意页面
- **当**: 点击导航或返回按钮
- **则**: 正确跳转到目标页面，无错误
- **验证类型**: 程序验证

### AC-4: 响应式布局正常
- **给定**: 用户在不同设备上访问
- **当**: 在手机、平板、电脑浏览器上查看
- **则**: 页面布局适配良好，元素不溢出
- **验证类型**: 人工判断

### AC-5: 部署成功
- **给定**: 完成所有功能开发
- **当**: 部署到 Cloudflare Pages
- **则**: 可以通过公网域名正常访问，所有功能正常
- **验证类型**: 程序验证
