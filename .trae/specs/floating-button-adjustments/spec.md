# 悬浮按钮优化 - 产品需求文档

## 概述
- **总结**: 优化悬浮发布按钮的位置，确保其显示在底部导航栏上方，同时确保按钮功能完整可用
- **目的**: 提升用户体验，避免按钮遮挡问题，确保发布功能正常使用
- **目标用户**: 邻里社区APP的所有用户

## 目标
- 调整悬浮发布按钮位置到底部导航栏上方
- 确保悬浮按钮的功能（发布动态、发布活动、发布互助）完整可用
- 确认首页没有重复的发帖按钮

## 非目标（范围外）
- 不修改悬浮按钮的UI设计样式
- 不修改底部导航栏的功能和样式
- 不添加新的发布功能

## 背景与上下文
根据用户反馈，当前悬浮发布按钮位置可能存在遮挡问题，且加号功能有丢失的情况。需要修复这些问题。

## 功能需求
- **FR-1**: 悬浮发布按钮位置调整 - 确保按钮位于底部导航栏上方足够的距离
- **FR-2**: 悬浮按钮功能验证 - 确保发布动态、发布活动、发布互助功能正常
- **FR-3**: 重复按钮检查 - 确认首页没有旧的发帖按钮

## 非功能需求
- **NFR-1**: 兼容性 - 在不同屏幕尺寸上都能正常显示和工作
- **NFR-2**: 性能 - 调整不影响应用性能

## 约束条件
- **技术**: 使用现有的Vue 3 + Vite技术栈
- **依赖**: 不引入新的第三方库

## 假设条件
- 现有的代码结构保持不变
- 底部导航栏的高度在移动端约为60px + safe-area-inset-bottom

## 验收标准

### AC-1: 悬浮按钮位置正确
- **前置条件**: 打开应用并进入首页或互助页面
- **操作**: 查看悬浮按钮位置
- **预期结果**: 悬浮按钮位于底部导航栏上方，不与导航栏重叠
- **验证方式**: 人工检查
- **备注**: 按钮底部距离导航栏顶部至少10px

### AC-2: 悬浮按钮功能完整
- **前置条件**: 打开应用并进入首页或互助页面
- **操作**: 点击悬浮按钮，分别测试发布动态、发布活动、发布互助选项
- **预期结果**: 每个选项都能正确跳转到相应的发布页面
- **验证方式**: 人工测试

### AC-3: 无重复按钮
- **前置条件**: 打开应用并进入首页
- **操作**: 浏览首页，检查是否有多个发布入口
- **预期结果**: 首页只存在一个发布入口（即悬浮按钮），没有其他发帖按钮
- **验证方式**: 人工检查

## 开放问题
无
