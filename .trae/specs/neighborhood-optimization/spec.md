# 邻里空间（客厅）优化 - 产品需求文档

## 概述
- **Summary**: 优化邻里空间（客厅）模块体验，在首页快捷入口添加客厅入口，精简客厅模块功能，保留核心体验
- **Purpose**: 提升用户访问客厅模块的便捷性，优化模块体验，去除冗余功能
- **Target Users**: 邻里社区APP所有用户

## 问题分析

### 1. 当前问题
- 首页快捷入口没有客厅入口，用户访问客厅不够便捷
- 客厅模块功能较多，需要精简优化核心体验
- 客厅模块标签栏：客厅、活动、小组、排行，部分功能可能冗余

### 2. 用户需求
- "健康打卡前面要有客厅的入口"
- "这个客厅模块还可以优化，去掉没用的功能和组件，保证核心体验"

## 优化方案

### 方案一：首页快捷入口优化
**目标**：在首页快捷入口添加"客厅"，放在健康打卡前面

**具体变更**：
- 首页 `quickActions` 数组调整顺序为：客厅、健康打卡、互助
- 客厅图标使用 🏠 或 🏘️
- 点击"客厅"跳转到 `/pages/neighborhood/index`

### 方案二：客厅模块精简优化
**目标**：优化邻里空间（客厅）模块，保留核心功能，去除冗余组件

**保留的核心功能**：
1. **客厅（lounge）**：社区聊天室、在线邻居 - 核心社交功能 ✅ 保留
2. **活动（activities）**：活动列表、参与活动 - 核心活动功能 ✅ 保留
3. **小组（groups）**：兴趣小组 - 社区兴趣功能 ✅ 保留
4. **排行（rankings）**：活跃邻居 - 激励功能，考虑精简

**优化策略**：
- 保留4个标签，但精简界面元素
- 优化视觉层次，让核心内容更突出
- 统一卡片样式，与整体UI风格一致
- 优化加载状态和空状态展示
- 移除过渡复杂的动画，保持简洁流畅

## 影响范围

### 受影响文件
- `src/pages/index/index.vue` - 首页快捷入口
- `src/pages/neighborhood/index.vue` - 客厅模块优化

## 新增需求

### 需求：首页添加客厅快捷入口
系统应该在首页快捷入口区域添加客厅入口，位于健康打卡之前。

**场景**：用户在首页看到快捷入口
- **WHEN** 用户打开首页
- **THEN** 看到快捷入口按顺序显示：客厅、健康打卡、互助
- **验证**：human-judgment
- **NOTES**：客厅图标使用 🏠 或 🏘️

## 修改需求

### 需求：优化客厅模块体验
**修改**：对邻里空间模块进行体验优化
**原因**：精简功能，提升核心体验

**具体变更**：
1. 优化顶部导航栏，保持简洁
2. 优化标签栏样式，与整体风格一致
3. 优化各个标签页的卡片样式
4. 统一间距和圆角规范
5. 优化过渡动画
6. 考虑是否保留排行榜（可作为可选优化）

## 移除需求
无（不删除任何核心功能）

## 约束条件
- **技术**：使用现有的 Vue 3 技术栈
- **设计**：与现有的 UI 风格保持一致
- **功能**：保留核心社交和活动功能
- **优先级**：先添加首页入口，再优化模块体验

## 假设条件
- 用户主要通过首页快捷入口访问各个功能模块
- 客厅、活动、小组是核心功能，需要保留
- 排行榜功能是辅助功能，可以考虑优化或保留
