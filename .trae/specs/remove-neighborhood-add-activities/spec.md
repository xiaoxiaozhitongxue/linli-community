# 删除邻里空间，保留独立活动页面 - 产品需求文档

## 概述
- **Summary**: 删除邻里空间页面，创建独立的活动列表页面，简化用户路径，提升核心体验
- **Purpose**: 以用户为中心，精简功能入口，让用户更直接地访问活动功能
- **Target Users**: 邻里社区APP所有用户

## 为什么需要这个优化

### 当前问题
1. **功能冗余**：邻里空间包含客厅、活动、小组、排行4个标签，但用户主要关注活动
2. **路径过长**：首页 → 邻里空间 → 活动标签，增加了用户操作步骤
3. **入口混乱**：首页既有"客厅"快捷入口，又有"查看更多"活动入口，用户可能困惑

### 用户需求
- "删除邻里空间，只保留活动页面"
- "由首页的更多活动点击跳转"
- "以用户为中心的理念重构"

## 优化方案

### 方案一：创建独立活动列表页面
**目标**：创建一个专门的活动列表页面 `/pages/activities/index.vue`

**具体变更**：
- 创建新页面 `src/pages/activities/index.vue`
- 包含活动列表展示（热门活动、近期活动）
- 支持活动筛选和搜索
- 提供发起活动按钮
- 优化活动卡片展示

### 方案二：修改首页跳转逻辑
**目标**：将首页"查看更多"指向新的活动页面

**具体变更**：
- 修改 `goToActivities()` 函数，跳转到 `/pages/activities/index`
- 从首页快捷入口移除"客厅"
- 保留"健康打卡"和"互助"两个快捷入口

### 方案三：删除邻里空间页面
**目标**：删除冗余的邻里空间页面

**具体变更**：
- 删除 `src/pages/neighborhood/index.vue`
- 更新路由配置，移除邻里空间路由
- 更新相关引用

## 影响范围

### 受影响文件
- `src/pages/activities/index.vue` - 新建活动列表页面
- `src/pages/index/index.vue` - 修改快捷入口和跳转逻辑
- `src/pages/neighborhood/index.vue` - 删除
- `src/utils/router.ts` - 更新路由配置
- `src/main.ts` - 更新路由定义

## 新增需求

### 需求：独立活动列表页面
系统应该提供一个独立的活动列表页面，展示所有活动。

**场景**：用户查看活动列表
- **WHEN** 用户点击首页"查看更多"
- **THEN** 跳转到独立的活动列表页面，展示热门活动和近期活动
- **验证**：human-judgment

### 需求：活动筛选和搜索
系统应该支持活动筛选和搜索功能。

**场景**：用户筛选活动
- **WHEN** 用户在活动页面选择筛选条件
- **THEN** 活动列表根据条件更新
- **验证**：human-judgment

## 修改需求

### 需求：首页快捷入口精简
**修改**：移除"客厅"快捷入口，只保留核心功能
**原因**：简化用户选择，聚焦核心功能

**具体变更**：
- 快捷入口只保留：健康打卡、互助
- "查看更多"跳转到独立活动页面

## 移除需求

### 需求：删除邻里空间页面
**原因**：功能冗余，用户主要关注活动，不需要复杂的标签切换
**迁移**：
- 活动功能迁移到独立的 `/pages/activities/index.vue`
- 其他功能（客厅、小组、排行）暂不保留，可后续按需添加

## 约束条件
- **技术**：使用现有的 Vue 3 技术栈
- **设计**：与现有的 UI 风格保持一致
- **功能**：保留核心的活动功能
- **优先级**：先创建活动页面，再修改跳转，最后删除旧页面

## 假设条件
- 用户主要关注活动功能，对客厅、小组、排行的需求较低
- 独立的活动页面能提供更好的用户体验
- 简化入口能降低用户认知负担
