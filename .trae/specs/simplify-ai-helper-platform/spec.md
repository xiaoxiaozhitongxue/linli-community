# 邻里社区APP - 简化版AI互助平台

## 概述

- **Summary**: 简化应用为AI互助核心平台，移除多余功能，新增消息通知（私信+群聊），优化底部导航
- **Purpose**: 聚焦核心功能，提升用户体验，降低使用门槛
- **Target Users**: 社区居民、志愿者、需要互助服务的用户

## 目标

- 简化底部导航栏：只保留「AI互助」「消息」「我的」三个入口
- 完善AI互助功能模块
- 新增消息通知系统（私信+群聊）
- 完善个人中心功能
- 屏蔽首页、邻里空间、创业模块的入口（保留代码，后续可恢复）

## 非目标（暂不包含）

- 首页功能模块
- 邻里空间模块
- 社区创业模块
- 支付功能
- 原生App打包

## 背景与上下文

当前问题：
1. 底部导航栏有5个入口，功能过于分散
2. 缺少消息通知系统
3. AI互助功能需要进一步打磨

## 功能需求

### FR-1: 简化底部导航栏
- 移除「首页」「邻里」「创业」三个入口
- 只保留「AI互助」「消息」「我的」三个核心入口
- 重新设计导航图标和样式
- 确保导航体验流畅

### FR-2: 完善AI互助功能
- 优化任务发布流程
- 完善任务匹配算法（展示）
- 优化任务详情页
- 添加任务状态管理
- 优化UI和交互体验

### FR-3: 新增消息通知系统
- **私信功能**：
  - 用户之间一对一聊天
  - 消息列表展示
  - 未读消息标记
  - 聊天详情页
  
- **群聊功能**：
  - 兴趣小组群聊
  - 群成员列表
  - 群消息展示
  - 群通知

### FR-4: 完善个人中心
- 优化个人信息展示
- 完善设置功能
- 优化我的任务（发布的+接受的）
- 添加消息通知设置

### FR-5: 屏蔽其他功能入口
- 屏蔽首页入口（保留代码）
- 屏蔽邻里空间入口（保留代码）
- 屏蔽社区创业入口（保留代码）
- 确保用户无法访问被屏蔽的页面

## 非功能需求

### NFR-1: 性能
- 页面加载 < 2秒
- 消息推送即时
- 流畅的交互动画

### NFR-2: 响应式设计
- 适配手机、平板、电脑
- 良好的触摸体验

### NFR-3: 代码组织
- 保留被屏蔽功能的代码（可恢复）
- 模块化开发

## 约束条件

- **技术栈**: Vue 3 + Vite + TypeScript + Tailwind CSS
- **部署**: Cloudflare Pages
- **消息系统**: 使用localStorage模拟（后续可接入真实后端）

## 假设条件

1. 使用Mock数据演示消息功能
2. 用户主要通过手机访问

## 验收标准

### AC-1: 导航栏简化
- **Given**: 用户打开应用
- **When**: 查看底部导航栏
- **Then**: 只显示「AI互助」「消息」「我的」三个入口
- **Verification**: `human-judgment`

### AC-2: AI互助功能完善
- **Given**: 用户在AI互助页面
- **When**: 发布任务、查看任务详情、接受任务
- **Then**: 所有功能正常工作，UI美观
- **Verification**: `human-judgment`

### AC-3: 消息通知功能
- **Given**: 用户点击消息入口
- **When**: 查看消息列表、发送私信、查看群聊
- **Then**: 消息功能完整，UI流畅
- **Verification**: `human-judgment`

### AC-4: 被屏蔽功能
- **Given**: 用户尝试访问被屏蔽页面
- **When**: 直接输入被屏蔽页面的URL
- **Then**: 重定向到AI互助页面或显示"功能暂未开放"
- **Verification**: `programmatic`
