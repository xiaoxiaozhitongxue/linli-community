# 界面布局优化和定位修复规范

## Why
当前应用存在以下问题：
1. 定位功能显示的位置与用户真实位置不一致，需要修复定位算法
2. 界面各组件对齐不统一，需要优化整体布局结构
3. 悬浮发布按钮（加号）出现透明效果，破坏了用户体验
4. 需要以产品经理思维系统性地确认所有控件的位置和功能

## What Changes
- [ ] 修复定位算法，确保根据真实经纬度计算最近社区
- [ ] 统一所有页面和组件的对齐方式，确保视觉一致性
- [ ] 修复悬浮发布按钮样式，确保不透明且位置正确
- [ ] 优化底部导航栏和其他核心组件的位置和间距
- [ ] 确保移动端和Web端布局都合理美观

## Impact
- Affected specs: UI协调性优化
- Affected code: 
  - src/pages/index/index.vue（首页定位和布局）
  - src/components/FloatingPublishButton.vue（悬浮发布按钮）
  - src/components/BottomTabBar.vue（底部导航栏）
  - src/styles/base.css（全局样式）

## ADDED Requirements

### Requirement: 精确的地理位置定位
系统应能够根据用户的真实GPS坐标，计算出距离最近的社区并显示

#### Scenario: 定位成功
- **WHEN** 用户允许定位且GPS可用
- **THEN** 显示距离用户最近的社区名称

#### Scenario: 定位失败
- **WHEN** 用户拒绝定位或GPS不可用
- **THEN** 显示默认社区名称并提示用户

### Requirement: 统一的组件对齐
所有页面和组件应遵循统一的布局规范，确保视觉一致性

#### Scenario: 页面布局
- **WHEN** 用户浏览各个页面
- **THEN** 所有组件应正确对齐，间距一致

### Requirement: 清晰的悬浮发布按钮
悬浮发布按钮应始终保持不透明，位置固定，不遮挡任何功能区域

#### Scenario: 悬浮按钮显示
- **WHEN** 用户在任意页面
- **THEN** 悬浮按钮清晰可见，不透明，位于底部导航栏上方

## MODIFIED Requirements

### Requirement: 底部导航栏位置固定
底部导航栏应在屏幕底部固定，高度为56px，背景为白色

### Requirement: 悬浮发布按钮位置
悬浮发布按钮应位于底部导航栏上方30px，右侧边缘20px

## REMOVED Requirements
无

## 产品经理思维：控件位置和功能确认

### 首页布局（从上到下）
1. **状态栏**（高度根据设备自适应）
   - 左侧：位置选择器（点击可选择社区）
   - 右侧：搜索入口

2. **横幅轮播**（高度约120px）
   - 全宽显示，自动轮播
   - 底部有指示点

3. **快捷入口**（高度约80px）
   - 两个入口：健康打卡、互助
   - 水平排列，间距16px

4. **热门活动**（高度约160px）
   - 标题行：标题 + 查看更多
   - 横向滚动卡片

5. **近期活动**（高度约200px）
   - 标题行：标题 + 查看更多
   - 纵向列表，最多3条

6. **邻里动态**（自适应高度）
   - 动态卡片列表
   - 支持下拉刷新、上拉加载

### 悬浮发布按钮
- **位置**：底部导航栏上方40px，右侧20px
- **尺寸**：56px × 56px圆形
- **样式**：橙色渐变背景，白色加号图标
- **功能**：点击展开菜单（发布动态、发布活动、发布互助）
- **层级**：z-index应高于内容但低于弹窗

### 底部导航栏
- **位置**：屏幕最底部，固定定位
- **高度**：56px
- **项目**：首页、活动、消息、我的
- **样式**：白色背景，顶部有分隔线

## 关键约束
1. 悬浮按钮不得遮挡底部导航栏
2. 所有文字和图标应清晰可见
3. 触摸目标至少44px × 44px
4. 响应式布局适配320px-1440px屏幕宽度
