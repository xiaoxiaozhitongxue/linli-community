# 首页UI修复与优化 - 产品需求文档

## Overview
- **Summary**: 修复首页定位显示和布局加载时的视觉问题，提升用户体验
- **Purpose**: 解决定位时文字闪烁、定位格式不优、页面加载时橙色区域错位等问题
- **Target Users**: 所有用户，特别是桌面端用户

## 问题描述

### 问题1：定位文字闪烁
- **现象**：定位时网页中间会闪一下黑字"定位到哪里哪里"
- **期望**：不应有任何闪烁效果，定位过程应该是静默的或只在状态栏显示

### 问题2：定位格式优化
- **现象**：当前定位显示格式可能包含省市区完整路径
- **期望**：定位显示改为"市+区"格式，简洁明了

### 问题3：首页橙色区域加载错位
- **现象**：每次加载首页时，橙色区域（顶部状态栏）会先错位，然后闪一下才正常
- **期望**：页面加载时不应有任何错位或闪烁，内容应直接以正确位置呈现

## Goals
- 移除定位时的文字闪烁效果
- 优化定位显示格式为"市+区"
- 修复首页橙色状态栏的加载错位问题
- 提升页面加载时的视觉稳定性

## Non-Goals (Out of Scope)
- 不改变定位功能的核心逻辑
- 不修改其他页面的样式
- 不改变移动端布局

## Background & Context
- 首页的顶部状态栏（橙色渐变区域）包含定位选择和搜索栏
- 定位功能在加载时和用户点击定位时触发
- 页面存在加载动画或布局重排导致视觉闪烁

## Functional Requirements
- **FR-1**: 定位时不应有任何文字闪烁或黑屏效果
- **FR-2**: 定位结果显示格式为"市+区"（如"深圳市-南山区"）
- **FR-3**: 首页橙色状态栏在页面加载时应直接显示在正确位置，无错位

## Non-Functional Requirements
- **NFR-1**: 页面加载时间不应增加
- **NFR-2**: 不引入新的布局问题
- **NFR-3**: 构建无错误

## Constraints
- **Technical**: Vue 3 项目
- **Dependencies**: 首页组件和样式

## Assumptions
- 定位数据可以获取到市区信息
- 页面加载闪烁可能是CSS动画或DOM渲染顺序问题

## Acceptance Criteria

### AC-1: 定位闪烁修复
- **Given**: 用户点击定位按钮
- **When**: 定位进行中
- **Then**: 页面上不应出现任何文字闪烁或黑屏效果
- **Verification**: `human-judgment`

### AC-2: 定位格式优化
- **Given**: 定位成功
- **When**: 显示定位结果
- **Then**: 显示格式为"市-区"（如"深圳市-南山区"）
- **Verification**: `human-judgment`

### AC-3: 首页状态栏无错位
- **Given**: 页面首次加载
- **When**: 页面渲染完成
- **Then**: 橙色状态栏直接显示在顶部正确位置，无任何错位或闪烁
- **Verification**: `human-judgment`

### AC-4: 构建成功
- **Given**: 代码修改完成
- **When**: 运行 `npm run build`
- **Then**: 构建成功无错误
- **Verification**: `programmatic`
