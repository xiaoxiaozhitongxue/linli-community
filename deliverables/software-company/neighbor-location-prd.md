**版本**: v1.0  
**创建人**: 产品经理  
**日期**: 2026-07-08  
**状态**: 草案

---

## 1. 产品目标

统一「邻里社区」小程序/Web app 各页面的定位交互体验，实现首页自动定位展示省市区，以及在发帖、互助、创建活动等场景中提供"自动定位+省市区下拉选择+详细地址可编辑"的完整位置选择能力，提升用户发布信息的便利性和位置数据的准确性。

## 4. 交互设计描述

### 4.1 首页（index.vue）

**当前问题**: 首页显示"点击定位"，用户必须点击弹出 location-picker 面板，在手动输入和自动定位两个 tab 中选择，再点"确认"才能完成定位。

**新交互**:

1. **进入首页时** → 自动触发 `startAutoLocate()`，NavBar 区域显示"定位中"（带旋转图标）。
2. **定位成功** → NavBar 直接展示 "xx市 - xx区"（如"上海市 - 浦东新区"），图标为 map-pin，不再弹面板。
3. **定位失败** → NavBar 展示"点击重试定位"，带警告图标；点击可重新触发自动定位。
4. **展开定位**（保留但简化）→ 点击定位文本展开一个**轻量级气泡卡片**（非全屏面板），仅展示当前城市/区域简要信息，不再包含手动输入和自动定位两个 tab 的复杂面板。
5. 删除旧的 location-picker 底部面板及其模板代码。

### 4.2 互助任务发布页（publish.vue）

**当前问题**: 仅有一个 `<input type="text" v-model="form.location">` 纯文本输入框。

**新交互**:

1. 将"服务地点"区域替换为 `LocationPicker` 组件。
2. 进入页面时自动定位，定位成功后：
   - 上方显示"国家/省/市/区"四个下拉选择框，自动选中定位结果对应的值。
   - 下方显示"详细地址"文本框，自动填入逆地理编码返回的详细地址。
3. 用户可任意修改下拉选择框（省/市/区联动），也可编辑详细地址文本框。
4. 提供"重新定位"按钮（小图标形式），点击后重新 GPS 定位刷新。
5. 如果定位失败或用户拒绝定位权限，省市区下拉框均未选中状态，由用户手动选择。

### 4.3 发动态页（create.vue）

**当前问题**: 点击位置区域弹出 modal，内含纯文本输入框。

**新交互**:

1. 点击位置区域（底部栏的"位置"或输入区的位置行）弹出 `LocationPicker` 组件（以底部弹窗形式）。
2. 弹窗内容与互助发布页一致的省市区下拉 + 详细地址编辑布局。
3. 进入弹窗时自动定位并填入。
4. 用户确认后，将完整位置文本（如"上海市浦东新区·张江路100号"）回填到页面展示。
5. 如果已选择过位置，再次点击进入弹窗时保留上次的值。

### 4.4 创建活动页（activities/create.vue）

**当前问题**: 点击"选择位置"弹出 modal，内含纯文本输入框。

**新交互**: 与发动态页（4.3）完全一致。

## 5. 数据模型建议

### 5.1 位置数据结构

```typescript
// 省市区地址组件化数据结构
interface LocationData {
  // 自动定位结果
  autoLocated: boolean

  // 省市区（支持联动下拉选择）
  country: string       // 国家，默认"中国"
  province: string      // 省/直辖市，如"上海市"
  city: string          // 市，如"上海市"
  district: string      // 区/县，如"浦东新区"

  // 详细地址
  detailAddress: string  // 街道门牌号/POI名称，用户可编辑

  // 原始坐标（留底）
  longitude?: number
  latitude?: number

  // 展示用完整地址（前端拼接）
  get fullAddress(): string {
    return `${this.country} ${this.province} ${this.city} ${this.district} ${this.detailAddress}`
  }
}
```

### 5.2 省市区数据源

### 5.3 现有 LocationResult 的兼容

现有的 `LocationResult`（`src/utils/location.ts`）仍然保留作为定位引擎的输出格式，新增的 `LocationData` 是面向 UI 展示和编辑的数据模型。从 `LocationResult` 到 `LocationData` 的映射：

| LocationResult 字段 | 映射到 LocationData |
|-------------------|-------------------|
| city | province → 提取省；city → 提取市 |
| district | district |
| address | detailAddress（主）+ 拼接省市区 |
| longitude/latitude | longitude/latitude |

### 5.4 表单提交字段

提交到后端时，位置信息统一为以下字段格式：

```typescript
interface LocationSubmitData {
  province: string      // 省
  city: string          // 市
  district: string      // 区
  address: string       // 详细地址
  longitude?: number    // 经度
  latitude?: number     // 纬度
}

// 展示文本 = province + city + district + "·" + address (如 "上海市上海市浦东新区·张江路100号")
// 注意: 直辖市场景下 province === city
```
