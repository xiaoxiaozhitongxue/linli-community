## 二、7 大优化方向（按优先级排列）

### 🔴 P0 · 信息流改为推荐排序

**现状**：首页 Feed 流按 `created_at DESC` 时间倒序，新旧内容混在一起，优质内容无法脱颖而出。

**市场标准**：
- **小红书**：双列瀑布流 + 千人千面算法推荐，新内容有冷启动流量池
- **Nextdoor**：AI 驱动的个性化排序，热门内容和本地新闻优先
- **抖音本地生活**：LBS + 兴趣标签混合推荐

**建议方案**：
```
热度分 = 点赞数×2 + 评论数×3 + 新鲜度衰减因子(1/(小时+2))
```
1. 后端 tasks/posts API 增加 `ORDER BY` 排序参数（hot/new 两种模式）
2. 首页 Tab 增加"热门"和"最新"切换
3. 中期可引入用户行为埋点（停留时长、点击率），逐步过渡到个性化推荐

**改动范围**：`/api/posts` (GET)、`pages/index/index.vue`（排序切换 UI）
**预估工作量**：⭐⭐（简单）

---

### 🔴 P0 · 信用/评价体系

**现状**：用户表有 `credit_score` 字段，个人主页展示但无详情页，无评价记录，用户之间无法互评。对比闲鱼的信用体系和评价系统差距明显。

**市场标准**：
- **闲鱼**：交易双方互评 + 评价标签（"描述相符""发货速度""沟通态度"）+ 信用分可视化
- **Nextdoor**：邻居推荐信（Recommendations），服务提供方有星级评价

**建议方案**：
1. 新建评价表 `reviews`（reviewer_id, target_id, score, content, tags, type(task/activity))
2. 互助任务完成后强制互评
3. 信用分详情页展示评价记录 + 评分分布
4. 个人主页增加"信誉"展示区

**改动范围**：新增 1 张数据表 + 1 个后端 API + 1 个前端页面
**预估工作量**：⭐⭐⭐⭐（较大）

---

### 🟡 P1 · 沉浸式 Feed 流体验

**现状**：首页 Feed 为单列卡片列表，每张卡片撑满宽度，信息密度低。对比小红书双列瀑布流，同屏信息量少 50%。

**市场标准**：
- **小红书**：双列瀑布流，图片优先，文字精简，浏览效率高
- **抖音**：全屏视频流，沉浸式上下滑动

**建议方案**：
1. 首页 Feed 改为**双列瀑布流**布局（CSS `columns` 或 Masonry 布局）
2. 帖子卡片改为图片优先、文字折叠（默认2行）
3. 列表页支持"大图模式"和"列表模式"切换
4. 活动/任务在首页 Feed 中以紧凑卡片嵌入，而非单独大卡片

**改动范围**：`pages/index/index.vue`（Feed 组件样式重构）
**预估工作量**：⭐⭐⭐（中等）

---

### 🟡 P1 · 本地发现与推荐

**现状**：没有"附近"或"同城"发现页，搜索是唯一的主动发现方式。社区维度未充分利用。

**市场标准**：
- **Nextdoor**：基于地理位置的 Feed（你附近的帖子优先）
- **抖音本地生活**：基于 LBS 的店铺/活动推荐
- **大众点评**：附近热门、距离排序

**建议方案**：
1. 互助 Tab 增加"附近任务"筛选（基于经纬度距离排序）
2. 首页 Feed 增加"来自你的社区"打标
3. 活动列表支持"距离最近"排序
4. 搜索页增加"附近热门"板块

**改动范围**：`/api/tasks` (增加距离参数)、多个列表页 UI
**预估工作量**：⭐⭐⭐（中等）

---

### 🟡 P1 · 实时推送通知

**现状**：所有通知需要用户主动打开消息页拉取，无浏览器推送。任务通知、评论回复等关键事件没有即时触达。

**市场标准**：所有竞品均支持 Web Push / App Push 推送

**建议方案**：
1. 接入浏览器 Push Notification API（Service Worker）
2. 关键事件（被接单、新评论、活动报名成功）触发推送
3. 消息列表增加红点实时更新（现已有基础，需改为轮询或 WebSocket）

**改动范围**：新增 Service Worker + `/api/notifications` 端点
**预估工作量**：⭐⭐⭐（中等）

---

### 🟢 P2 · 短内容/状态动态

**现状**：帖子是全格式的（文字+多图），缺少轻量级的"状态"或"随手拍"模式。

**市场标准**：
- **微信朋友圈**：图文+小视频
- **小红书**：笔记（图文）和视频双轨
- **Nextdoor**：快速提问、投票、推荐等轻量发帖模板

**建议方案**：
1. 发帖界面增加"提问""投票"等轻量模板
2. 支持单张图片快速发布（简化发帖流程）
3. 社区公告/投票功能（如"周末集市是否延期？"）

**改动范围**：`pages/post/create.vue`、数据表加 type 字段
**预估工作量**：⭐⭐⭐（中等）

---

### 🟢 P2 · 社区创业体验闭环

**现状**：商品无购买流程（已移除），只有"联系店主"。缺少在线沟通、订单管理闭环。

**市场标准**：
- **闲鱼**：IM 即时聊天 + 担保交易
- **小红书**：笔记种草→店铺下单闭环

**建议方案**：
1. 集成私信功能到商品详情页（一键私聊店主）
2. 订单状态管理完善（待付款→待发货→待收货→已完成）
3. 商品收藏/关注店铺功能

**改动范围**：`pages/business/` 多个页面、私信集成
**预估工作量**：⭐⭐⭐⭐（较大）

## 五、与主流竞品的核心差距雷达图

```
               信息流推荐
                   │
               🔴   │
     商业化 ◄───────┼───────► 社交互动
               🟡   │   🟡
                   │
    本地发现 ◄───────┼───────► 实时通知
               🔴   │   🟡
                   │
               🔴   │
             信用体系
```

**图例**：🔴 差距大  🟡 有差距  🟢 基本持平

**邻里社区相较于 Nextdoor/小红书的优势**：
- ✅ 功能全面度（50 个页面覆盖社区核心场景）
- ✅ 互助任务闭环（发布→接单→完成）
- ✅ 老龄化关怀（养老服务专版）
- ✅ 免费+无广告（相比竞品商业化程度低）

**核心短板**：缺算法推荐、缺信任体系、缺实时推送
