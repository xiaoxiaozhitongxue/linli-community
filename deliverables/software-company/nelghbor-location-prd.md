# 邻里社区APP - 定位功能重构 PRD

## 产品目标
简化首页定位交互，统一发帖/互助/活动三个发布页面的定位选择体验，支持自动定位+省市区下拉选择+详细地址编辑。

## 用户故事
- 作为用户，我希望打开首页就能看到自己所在的省市区，不需要手动点击定位
- 作为用户，我在发帖/发布任务/创建活动时，系统自动定位并填好省市区和地址，我只需微调即可
- 作为用户，当我需要改变位置时，可以通过下拉框自由切换省市区

## 需求池

### P0 - 核心需求
1. **首页自动定位展示**：页面加载自动定位，NavBar直接显示"省-市-区"格式，移除点击展开的location-picker面板
2. **三个发布页面定位集成**：ai-helper/publish, post/create, activities/create 三个页面集成统一的位置选择组件
3. **自动定位填充**：进入发布页面时自动定位，自动填充国家、省、市、区、详细地址
4. **省市区下拉选择**：省/市/区分别用原生select下拉框，数据从内置省市区分级数据读取
5. **详细地址可编辑**：详细地址用文本输入框，自动填入后允许用户修改

### P1 - 重要功能
6. **省市区级联数据**：内置一份中国省市区三级数据（重点覆盖目标城市）
7. **定位结果匹配**：自动定位的city/district自动匹配到对应的下拉选项

### P2 - 后续优化
8. **定位状态反馈**：定位中显示加载状态，定位失败显示提示
9. **国家字段**：展示"中国"作为国家字段

## 交互设计

### 首页
- NavBar区域：自动定位后显示"上海市-浦东新区"格式，仅有展示功能，无点击交互
- 移除location-picker底部面板所有相关模板、样式和逻辑
- 定位中显示"定位中..."，定位失败显示"定位失败"

### 发布页面（统一交互）
每个发布页面在表单中嵌入统一的位置选择模块：
- 显示5行字段：
  1. 国家：固定显示"中国"（不可修改）
  2. 省：`<select>` 下拉框，自动选中定位结果
  3. 市：`<select>` 下拉框，自动选中定位结果
  4. 区：`<select>` 下拉框，自动选中定位结果
  5. 详细地址：`<input>` 文本输入框，自动填入定位地址
- 提交表单时：location 字段格式化为 "省-市-区 详细地址"
