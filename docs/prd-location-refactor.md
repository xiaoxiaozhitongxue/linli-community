# PRD：邻里社区 APP — 定位功能重构

## 1. 项目信息

- **Language**: 中文
- **Programming Language**: Vue 3 (Vite) + TypeScript（当前项目已有技术栈）
- **Project Name**: `location_refactor`
- **原始需求**：
  - 首页不需要弹出定位面板让用户输入或选择，直接自动定位展示省市区即可。
  - 发帖/动态/互助/活动页面需要在自动定位后展示国家、省、市、区/县、详细地址；省市区使用下拉选择框允许修改，详细地址自动填入但允许编辑。

---

## 2. 产品定义

### Product Goals (3 个)

1. **零摩擦体验** — 首页定位无需任何用户操作，页面加载后自动完成并直观展示
2. **精准可控的地址输入** — 发布页面自动定位基础地址，同时提供省市区级联选择和详细地址编辑能力
3. **统一可维护的定位组件** — 所有发布页面复用同一套省市区选择器组件，减少重复代码

### User Stories (4 个)

1. As a **普通用户**, I want **打开首页就看到当前所在省市区** so that **我知道社区定位已经正确且不需要额外操作**
2. As a **发布任务的用户**, I want **发布互助任务时位置自动填入省市区+详细地址** so that **省去手动输入的麻烦**
3. As a **修改变更位置的用户**, I want **能通过下拉选择框修改省市区** so that **我的地址不在定位地点时也能正确标注**
4. As a **活动组织者**, I want **创建活动时能自动定位并自由修改详细地址** so that **活动地点信息准确且方便**

#### 发布页面（三个页面统一改造）

```
┌─────────────────────────────────┐
│  ← 发布互助                      │
├─────────────────────────────────┤
│  [标题]  [描述]  [类型选择] ...   │
│                                 │
│  📍 位置                        │
│  ┌──────────────────────┐       │
│  │ 国家：中国（固定显示） │       │
│  │ 省：   [上海市 ▼]     │       │  ← 原生 select
│  │ 市：   [上海市 ▼]     │       │  ← 原生 select
│  │ 区/县：[浦东新区 ▼]   │       │  ← 原生 select
│  │ 详细地址：            │       │
│  │ [xx路xx号xx小区xx室]  │       │  ← 可编辑输入框
│  └──────────────────────┘       │
│                                 │
│  [报酬]  [AI推荐] ...            │
│  [        发布任务        ]      │
└─────────────────────────────────┘
```

- 新增 `LocationPicker` 组件（或页面内联），包含：
  1. 国家字段：固定显示"中国"，不可编辑
  2. 省字段：`<select>` 下拉框，选项预设
  3. 市字段：`<select>` 下拉框，根据省筛选
  4. 区/县字段：`<select>` 下拉框，根据市筛选
  5. 详细地址：`<input>` 文本输入框
- 页面加载时序：先自动定位 → 自动匹配省市区下拉值 → 自动填入详细地址
- 用户可随时修改任何字段
